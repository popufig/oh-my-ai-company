## 方法上学到的

1. **同为 enterprise AI，高价值比较不一定是同赛道竞品。** Harvey 和 Sierra 不是直接竞品，但都代表“强专业垂直 + 企业信任 + agent workflow + 大客户 GTM”。这种横向对照有助于提炼 enterprise agent 共性。

2. **法律行业社区反馈比泛科技社区更重要。** Reddit r/legaltech / LawNext 的问题质量很高：Claude 替代、法律数据源、hallucination、privilege、ethical walls、pricing、frontier model moat。这些问题直接指向产品护城河。

3. **Similarweb 里的 similar sites 要分层。** Harvey 的类似网站里 Legora/Paxton/Spellbook/Darrow/Eve/Clio 是法律 AI/LegalTech；Sierra/Decagon/OpenAI/Claude/Cursor 是 enterprise AI/agent 受众相邻。不能一张表里全叫竞品。

4. **渠道页存在口径差异。** Harvey overview 和 traffic source detail 的渠道百分比不一致，但绝对量、Top sources 和搜索页能互相补充。写入时要标口径，不要把一个百分比当唯一事实。

5. **付费搜索能暴露真实竞争。** Harvey 付费搜索重点买 `legora`，比官方竞品表更能说明它在市场上把谁当对手。

## 工具/流程问题

- Similarweb + screenshot/eval 不能并发跑，同一个 tab 容易 `target closed`。以后同 tab 操作全部串行。

- Gabe 的 X 账号没有在 bio 中写 Harvey，暂不作为 confirmed touchpoint。

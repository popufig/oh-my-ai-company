# Agent 原生上下文工作空间
Agent 原生上下文工作空间不是在旧文档工具旁边增加一个聊天框，而是把组织的知识与工作状态改造成 agent 可直接读取、修改、版本化和执行的底座。

## 典型构成

- Markdown、CSV、HTML、代码与目录树等开放文件结构。
- 人和 agent 共同读写的持久 workspace。
- 显式 Rules、Memory、Skills、Integrations 与 Automations。
- Workflow/任务状态与人类审批。
- 可直接交付的 MiniApp、dashboard、页面和报告。

## 为什么重要

模型能力逐渐同质化后，差异更多来自 agent 能看到什么、状态如何持续、能否安全行动，以及产物能否进入下一步流程。上下文层一旦积累组织规则和历史，就可能形成比单次生成更强的迁移成本。

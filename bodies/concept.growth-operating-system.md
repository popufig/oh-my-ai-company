# 增长操作系统

一句话：把增长从一堆分散任务，变成可执行、可复用、可闭环的工作流系统。

这个产品模式来自 [[company.ploy]]。它不是单纯做网页生成，也不是单点 SEO 工具，而是把 web coding、SEO/AEO、ABM 页面、内容页、访客识别、发布检查、外部系统集成串成一套增长流程。

这个 concept 目前还是 candidate，因为主要来自 Ploy 一个样本。后续如果看到更多公司也在做“增长 playbook + 外部系统打通 + 自动执行闭环”，再把置信度提高。

判断一个产品是不是增长操作系统，可以问：它只是生成素材，还是把增长动作从发现机会、生成页面/内容、发布、追踪反馈到下一轮优化都串起来？

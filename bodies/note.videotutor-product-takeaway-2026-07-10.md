# VideoTutor takeaway：教育 agent 的输出是教学过程，不是答案

[[company.videotutor]] 对我们最有价值的启发是：**agent 的输出形态可以从“答案”升级为“过程”。**

大多数 AI tutor / homework helper 的心智是：学生问问题，系统给答案或步骤。VideoTutor 的心智更进一步：学生问问题，系统生成一段可视化、可打断、可追问、能记忆上下文的教学视频。它卖的不是“我知道答案”，而是“我能像老师一样把这个概念讲懂”。

这对 agent 产品有几个启发：

1. 输出媒介本身可以成为产品壁垒。文本答案会被通用大模型快速吞掉，但数学动画、语音讲解、实时追问、记忆个性化的组合更垂直。
2. 垂类 agent 要找“任务闭环”。VideoTutor 的闭环是：题目输入 -> 准确解析 -> 动画讲解 -> 追问 -> 记忆 -> 考试提分。
3. GTM 可以跟用户生成内容绑定。它不像 B2B agent 靠 LinkedIn/HN/PH，而可能靠学生 TikTok 分享、家长群、校园 ambassador 起量。
4. 真实性验证很关键。教育产品的宣传数字容易漂亮，但最终要看真实题目准确率、延迟、学习效果和留存。

我会把 VideoTutor 归为：**education agent / generative instruction / AI video tutor / STEM visualization**。

它不是 AI employee，但它很适合作为我们理解“agentic product form”的样本：agent 不只是操作软件，也可以实时生成一个可交互的教学体验。

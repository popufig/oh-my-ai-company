# 从 Viktor 投资人网络挖出的相邻公司地图

这次不是为了把 Accel / Leonis / Bek 的 portfolio 全部抄一遍，而是沿着 Viktor 的投资逻辑找“同一套判断下还押了什么”。结果比较清楚：三家投资方分别代表三种相邻路径。

## 1. Accel / Zhenya：AI builder + enterprise agent + vertical autonomous agent

[[person.zhenya-loginov]] 在 Accel 侧和 Viktor 关系最强。除了 [[company.viktor]]，Accel/Zhenya 还挂着几个相邻标的：

- [[company.lovable]]：AI web app builder。Accel 2025 年领投 $200M Series A，文章 [[source.accel.lovable-series-a-2025-07-17]] 说 Lovable 让“other 99%”也能造 full-stack software。这个和 Viktor 的共同点是：AI 不停留在 chat，而是进入生产系统。
- [[company.synthflow-ai]]：AI voice agent builder，Series A in 2025，partner 是 Zhenya。它代表 voice agent / enterprise communications。
- [[company.solda-ai]]：Autonomous sales agents for Fintech and Banking，seed in 2024，partner 是 Zhenya。它代表垂直 sales agent。

判断：Accel/Zhenya 的线不是单纯“AI agent”，而是 AI 怎么重新分配工作能力：非技术人造软件、语音 agent 执行沟通、sales agent 执行业务流程、Viktor 在 Slack/Teams 里做团队员工。

## 2. Leonis：agent thesis 最系统

[[investor.leonis-capital]] 的线最像“agent thesis portfolio”。它不只是投了 Viktor，还在 portfolio page 和研究文章里系统铺了 agent 方向。

关键相邻标的：

- [[company.kylon]]：AI-native workspace where agents and humans collaborate as peers in one chat。
- [[company.general-analysis]]：production agents 的 red teaming 和 runtime controls。
- [[company.scalestack]]：AI-powered GTM platform，autonomous revenue engines。
- [[company.paysponge]]：让 AI agents hold money / conduct transactions 的 fintech infra。
- [[company.introspection]]：持续改进 production agents 的 feedback/practices 系统。
- [[company.qualgent]]：mobile QA agents。

Leonis 的研究文章 [[source.leonis.ai-agents-future-software]] 提出 Software 3.0，强调 specialized tools、multi-agent systems、agent frameworks、strategic reasoning。它投 Viktor 的逻辑和这些标的是同一套：agent 要真实进入生产，就需要框架、工具、上下文、安全、支付、评估、垂直任务。

判断：如果我们要找下一批“Viktor 周边公司”，Leonis portfolio 是很好的种子池，尤其是 agent infra 和 vertical agents。

## 3. Bek：从 RPA/企业自动化到 AI 进入低自动化行业

[[investor.bek-ventures]] 的线和 Leonis 不一样。Bek 更像从企业自动化/垂直行业 ROI 出发。

相邻标的：

- [[company.uipath]]：Bek portfolio 称它是 world leading agentic automation platform，RPA + AI + process mining + cloud services。
- [[company.interloom]]：Navigation System for Work，map how teams actually work，为 operational teams 和 AI agents 提供 next steps/context。
- [[company.lace-ai]]：AI sales/revenue intelligence for home services。Bek 文章 [[source.bek.lace-ai-thesis-2025-05-07]] 强调 AI 的机会不只在技术行业，也在 traditionally lagged in automation 的市场，关键是直接带来收入和 ROI。

判断：Bek 投 Viktor 的背景可能不是“又一个 AI hot deal”，而是它本来就关注 enterprise workflow automation、work context、垂直行业 AI ROI。Viktor 把这些组合进 Slack/Teams AI employee 形态。

## 4. 下一步优先级

如果要从这张图里挑下一批深挖对象，我建议：

1. [[company.interloom]]：和 Viktor 在“理解团队如何工作 / 给 agent 提供 context”上最贴近。它可能不是竞品，而是 agent work graph / context layer。
2. [[company.solda-ai]] 或 [[company.lace-ai]]：看 vertical sales agent 如何定价、交付 ROI、进入传统行业。
3. [[company.kylon]]：agent-human workspace，与 Viktor 的 Slack/Teams 工作入口相邻。
4. [[company.general-analysis]] / [[company.introspection]]：如果我们关心 agent 生产化，安全、评估、runtime control 是必挖基础设施。
5. [[company.lovable]]：不是 Viktor 竞品，但和 Accel/Zhenya 的 AI production system thesis 高度相关，也能和我们之前的 Superset/Ploy/web coding 线连接。

## 5. 对 Viktor 的反向理解

这张投资网络强化了一个判断：Viktor 不是孤立公司。它站在三条趋势交叉点：

- Enterprise automation 从 RPA/workflow 迁移到 agentic automation。
- AI coding / app building 降低软件生产门槛。
- Agent 从 demo 进入生产，需要 context、tools、security、payments、evaluation。

所以分析 Viktor 不能只看 Slack bot 本身，要看它是否能变成企业 agent operating layer：既拿到工作入口，又拿到上下文和工具权限，还能可靠闭环执行。

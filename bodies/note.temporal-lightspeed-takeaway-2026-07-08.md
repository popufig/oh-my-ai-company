# Temporal-Lightspeed takeaway

Lightspeed’s Temporal post is valuable because it turns a mature distributed-systems category into an AI-agent investment thesis.

Their argument:
- Teams can build impressive AI demos quickly.
- Production fails when agents take multiple steps, make intermediate tool calls, and run for minutes/days/months.
- Error rates compound over longer horizons; retries duplicate work; state gets lost; engineers manually restart autonomous processes.
- Temporal lets teams move from “cool demo” to recoverable, auditable, consistent production experience.

This helps structure the Lightspeed map:
- [[company.exa]] = agents need fresh knowledge/retrieval.
- [[company.temporal]] = agents need durable execution/recovery.
- [[company.paid]] = agents need monetization/value proof.

That trio is a useful “agent substrate” frame for future portfolio scans.

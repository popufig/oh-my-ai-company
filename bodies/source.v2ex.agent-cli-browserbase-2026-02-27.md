# V2EX: AI/Agent 让 CLI 再次伟大？

证据等级：S3（中文开发者社区讨论）

这条讨论把 Browserbase / browser-use / Playwright + AI 放在“教 AI 点击网页”的方向里，并提出一个重要反思：GUI 是为人设计的，程序和 agent 更适合 CLI/API/MCP。

可用信号：
- 浏览器自动化被认为必要，但“截图、视觉识别、模拟点击、等待渲染、再截屏”的链路慢、不确定、易碎。
- 更理想的方向可能是 API/MCP；但现实世界大量能力仍只有网页入口，所以浏览器自动化仍有空间。
- 对 Browserbase 的判断不应是“浏览器万能”，而应是“在没有 API/MCP、必须进入网页时，把这条脆弱链路工程化”。

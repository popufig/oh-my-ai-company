# Conviction 投资机构调研流程记录（2026-07-21）

## 为什么用它做第三个校准样本

[[investor.first-round-capital]] 验证精品早期机构，[[investor.andreessen-horowitz]] 验证多基金巨型平台；Conviction 用来检验个人品牌强、团队小、主题集中的 AI-native 机构。旧卷宗已有 portfolio、Embed 和 Sarah Guo 线索，因此本轮重点不是从零重写，而是校正 2025–2026 的资本与组织变化。

## 实际路径

1. 先查现有图谱，确认已有 13 条 portfolio relation、13 条机构 source、4 个触点，但只有 Sarah 一位结构化人物。
2. 用官网当前 People/Portfolio/Projects 校准团队、公开公司和分发入口。
3. 用 SEC Form D 拆出 Fund II、Expansion I、Select I，分别保留 offering、sold、first sale 与 related persons。
4. 用 Sola、Latent 官方 launch 文章识别 Sarah + Pranav 的公开署名关系，不把全量 portfolio 都猜成个人项目。
5. 用 MoE、Embed、No Priors、Commit 观察小团队如何通过人才、内容和 cohort 放大机构网络。
6. 用中文报道检查 Sarah 个人品牌与 HeyGen 美国化叙事如何覆盖机构本身。

## 新发现

- 小机构也不能只建一个 fund：Expansion 与 Select vehicle 会改变 follow-on 和机会投资的理解。
- 小团队更适合近全量扫描，但仍只把投资侧关键人物建为对象；Talent/EA 先留在官网 source。
- 个人品牌既是 sourcing/distribution 资产，也是 key-person risk。
- 官网 portfolio 适合发现候选；官方 launch/byline、Form D 和 company financing 才适合建立更强的轮次与人物关系。

## 工具与证据边界

Pinix 本轮处于 command channel 假在线状态，browser/site 空输出被排除在证据外；官网、SEC、LinkedIn 与中文材料改用可验证网页抓取。没有用空输出推断社区无讨论，也没有生成页面截图；正文只使用官网原始 logo。

## 对 SOP 的回焊

本轮确认 SOP 需要按机构规模裁剪，而不是每家执行同样的工作量：小型 AI-native 机构可以扫描完整投资侧团队和公开 portfolio，但必须额外研究个人品牌、cohort、内容与人才网络；大型机构则先拆 stage/vertical/vehicle，再做 bounded expansion。

本轮完成后，[[method.investor-research-sop-v0]] 已具备精品早期、多阶段平台、小型 AI-native 三类样本，可以进入 active v0；后续批量运行仍保留逐轮反思与修订。

# Duvo 产品 takeaway

[[company.duvo]] 对我们最有价值的地方：它把 agent 从横向“能做很多事”压进了 retail/FMCG/CPG 的具体运营工作，并且把产品包装成 process truth + mapped compliance + real-stack execution。

和 [[company.viktor]] / [[company.dust]] / [[company.interloom]] 的关系：

- Viktor：AI employee / workplace execution surface。
- Dust：AI Operator platform / multiplayer agent workspace。
- Interloom：Context Graph / corporate memory / operational routing layer。
- Duvo：vertical AI workforce / retail operations run layer。

产品启发：

1. **Vertical agent 的价值来自 messy workflow，不是漂亮 demo。** Duvo 明确说 happy path 只有一部分，真正价值在 SAP、portals、spreadsheets、email、phone calls、missing data、exceptions、human approvals 这些脏活里。
2. **Process truth before automation。** 它不是先卖 builder，而是先 map real process、fix broken process、再 run approved work。
3. **Production agent 需要 runtime。** Duvo developer page 反复强调 queues、runs、approvals、files、schedules、webhooks、MCP、audit-ready history。
4. **定价贴业务单元。** 它不按 token 或 seat，而按 process run：invoice processed、price change、SKU recovered。这比模型用量更贴近客户购买结果。
5. **Machine-readable GTM 值得学。** Duvo 有 pricing.md、compare.md、developers.md、agent-card.json、MCP card、OpenAPI、CLI，这明显是在让 agents/search/technical evaluators 更容易理解它。

风险/待验证：

- 公开客户结果多来自官网/官方披露，需要 case study 原文交叉验证。
- Similarweb 暂未拿到有效 public snapshot。
- 它现在是否更多是 FDE/实施型业务，还是可规模化产品，仍需看客户案例、demo 和招聘。

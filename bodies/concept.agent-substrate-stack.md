# Agent substrate stack

Production agents require more than model capability. The stack underneath agents is splitting into substrate layers. Lightspeed's portfolio is a useful map because it includes companies across several layers:

- Retrieval / fresh context: [[company.exa]] and [[concept.agent-search-retrieval-layer]].
- Durable execution / recovery / orchestration: [[company.temporal]] and [[concept.agent-durable-execution-layer]].
- Monetization / value proof / billing: [[company.paid]] and [[concept.agent-monetization-infra]].
- Security / validation / coding risk: [[company.endor-labs]] and AURI-related sources.
- Production debugging / DevOps multi-agent workflows: [[company.resolve-ai]].
- Enterprise workflow / AI services: [[company.reindeer-ai]].
- Data supply / rights-cleared multimodal training data: [[company.luel]].

Why this matters:
- Agent demos are cheap; production substrate is expensive and sticky.
- The higher-value substrate layers are where repeated agent work produces state, evidence, cost, value, or risk that enterprises need to control.
- Lightspeed appears to invest both in direct agent applications and in the infrastructure that lets agents become reliable business systems.

Open questions:
- Which layers become independent companies versus features inside agent platforms?
- Do successful agent app companies vertically integrate retrieval/execution/billing, or buy these layers?
- Which investor networks repeatedly co-invest in each substrate layer?

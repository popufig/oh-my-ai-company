# Agentic SDLC 团队协作层

一句话：AI 写代码变快后，真正的新瓶颈不一定是“生成代码”，而是团队如何规划、并行派发 agent、评审、预览、合并和上线。

这不是一个普通 tag，而是一个可复用的产品模式。它用来识别那些站在 Cursor / Copilot / Claude Code 之上的产品：不是再做一个单人 coding assistant，而是做“多人 + 多 agent + 真代码库”的协作层。

已观察到的例子：[[company.lightsprint]] 用 Visual Plan Mode、Parallel Cloud Agents、PR Preview Environments 把自己定位成 collaborative product development platform；[[company.superset]] 的 agent cockpit / IDE 也部分接近这个方向，但更偏 agent workbench。

后续看同类产品时，可以问：它是在提高单个工程师写代码速度，还是在解决团队如何协调 agentic work？如果是后者，就可以挂到这个 concept。

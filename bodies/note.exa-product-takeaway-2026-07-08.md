# Exa product takeaway

Exa is not merely “Google search API for agents.” The stronger thesis is that agents need a new retrieval substrate:

- Human search wants ranked pages and a UI.
- Agent search wants structured, fresh, cited, token-efficient evidence that can be called repeatedly inside workflows.
- Coding agents and GTM agents have different latency/quality profiles, so the product needs modes instead of one universal search endpoint.

What makes Exa strategically interesting:
- Official scale claims: 400K developers, 5K companies, 500B+ URLs tracked, $2.2B valuation.
- Customer logos include Cursor, Cognition, HubSpot, OpenRouter, Monday.com.
- GitHub surface is strong: MCP repo 4.7K stars; company researcher 1.5K stars; 102 repos in org.
- Docs are written partly for coding agents, including a “common LLM mistakes” section. This is product distribution through agent-readable docs.
- HN launch had real discussion and founder replies, with useful limitation disclosure: people/companies/papers/high-quality written content are strong; products/ecommerce/auth-gated/non-English content were weaker at launch.

Risk:
- The category may fragment by vertical.
- High-volume search costs matter; HN comments explicitly pushed back on pricing.
- Web crawling at agent scale may trigger anti-scraping and publisher/site-owner resistance.

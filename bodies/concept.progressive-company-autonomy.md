渐进式公司自治不是“一键让 Agent 接管公司”，而是先让 Agent 处理低风险、重复、可审计的工作，再随着可靠性和权限控制提高，把更多职能变成持续运行的 standing workflow。

判断一个产品是否真正支持渐进自治，至少看：

1. Agent 是否有持续目标、记忆和 heartbeat，而非只响应 prompt；
2. 动作按风险分级，哪些逐次批准、哪些 standing authorization；
3. 失败能否停机、升级、回滚并留下 audit trail；
4. 人类仍负责哪些战略、关系和异常判断；
5. “自治百分比”是否说明分母、时间窗和动作类型。

## 我的产品判断

1. **Polsia 把 AI employee 推到“公司托管运营商”这一层。** 它不只给岗位和聊天界面，而是替用户选择 infrastructure、执行外部动作、经手广告与支付。真正的产品边界是控制权和责任，不是 Agent 数量。
2. **它最强的设计是产品、GTM 和融资同构。** Live dashboard 是产品 UI，也是 data room 和传播素材；生成公司又自动产出 X 内容，增长本身持续制造下一轮增长叙事。
3. **规模是真的，headline 需要拆。** 当前可复算订阅年化约 $5.07M，混合 run rate 约 $8.53M。把广告代投和一次性包年化不等于造假，但叫 ARR 会让外界误读稳定性。
4. **“零员工”更像组织设计实验加叙事约束。** 它迫使 Ben dogfood，也依赖四家合作公司的工程师、基础设施 vendor 和人类最终判断。这里值得学的是约束，不是字面 headcount。
5. **最大的产品问题是可靠性与责任不对称。** Agent 能发信、投广告、收退款，用户却承担成本与合规；社区失败样本与官方 52.1% 30-day paid churn 同向，后续应追踪 accepted outcome 与 cohort，而不是更多 created companies。

## 对 founder 的启发

- 实时、可复算的 operating receipt 比普通案例页更有传播力，但指标定义必须经得住反向审计；
- 争议定位能快速拿 attention，长期仍要靠结果质量把 attention 变成 retention；
- Agent 产品的商业模型可以同时覆盖 subscription、usage、transaction 和 outcome，但每一层都带来新的 liability；
- “零员工”可以是 dogfooding 约束，不能把合作方工程、供应商服务和人类信任隐形化；
- 对 agent infra 创业者，Polsia 也证明了 Browserbase、Sapiom、Blaxel、AgentMail 等底层能力能通过一体化成品进入普通创业者市场。
